# GitHub 安全规则

## 一、绝对不能上传的内容

以下内容不能上传到 GitHub：

1. API Key
    
2. Cookie
    
3. 账号密码
    
4. 数据库密码
    
5. `.env` 文件
    
6. 平台登录凭证
    
7. 自动发布工具的账号配置
    
8. 私钥、token、access token
    

## 二、可以上传的内容

可以上传：

1. `.env.example`
    
2. 示例配置
    
3. 不包含真实账号的模板
    
4. 使用说明文档
    
5. 公开资料
    
6. 学习笔记
    

## 三、如果不小心上传了怎么办？

第一步：立刻去对应平台后台删除或重置密钥。  
第二步：从 GitHub 仓库中删除敏感文件。  
第三步：检查 Git 历史记录里是否还存在敏感内容。  
第四步：更新 `.gitignore`，避免再次上传。  
第五步：重新提交修复记录。

## 四、我的使用原则

1. 所有真实密钥只放本地
    
2. 所有示例配置用 `.env.example`
    
3. 公开仓库不放账号信息
    
4. 自动发布类项目只先看结构，不直接运行
    
5. 涉及 Cookie、账号、平台登录的内容，一律先确认风险
