# 需求处理流程

这个文档用于规定：当我有一个新需求时，应该如何在仓库里查找资料、调用 Skill、生成结果，并沉淀成可复用内容。

## 一、需求进入

所有还没有整理好的需求，先放到：

`inbox/`

例如：

- 一个短视频选题
    
- 一个直播主题
    
- 一个提示词想法
    
- 一个 AI 工具需求
    
- 一个学习问题
    

## 二、判断需求类型

### 1. 短视频相关

优先查看：

- Skill/短视频选题分析SKILL.md
    
- prompts/短视频脚本模板.md
    
- prompts/短视频脚本生成提示词.md
    

### 2. 直播相关

优先查看：

- Skill/直播选题分析SKILL.md
    
- prompts/直播预告文案提示词.md
    

### 3. GitHub 学习相关

优先查看：

- docs/GitHub学习笔记.md
    
- docs/内容检索索引.md
    
- logs/内容查找练习.md
    

### 4. 开源项目学习相关

优先查看：

- docs/GitHub开源项目学习清单.md
    
- docs/baoyu-xhs-images拆解.md
    

## 三、查找资料

查找顺序：

1. 先看 README.md
    
2. 再看 docs/内容检索索引.md
    
3. 用 Obsidian 搜索关键词
    
4. 用 GitHub 的 Go to file 搜文件名
    
5. 最后再用 GitHub 顶部搜索框
    

## 四、执行任务

根据需求选择对应 Skill 或 prompt。

例如：

短视频选题需求：

1. 打开 Skill/短视频选题分析SKILL.md
    
2. 打开 inbox 里的原始需求
    
3. 把 Skill + 需求一起给 GPT
    
4. 得到分析结果
    
5. 保存到 docs
    

## 五、沉淀结果

处理完后，根据内容类型归档：

- 分析结果 → docs
    
- 可复用提示词 → prompts
    
- 可复用工作流 → Skill
    
- 操作过程 → logs
    
- 临时想法 → inbox
    

## 六、提交 GitHub

每次完成一个清晰任务后：

1. GitHub Desktop 查看变化
    
2. Summary 写本次修改
    
3. Commit
    
4. Push
    
5. 必要时创建 Pull Request
    
6. 更新 Issue / Project / Milestone
