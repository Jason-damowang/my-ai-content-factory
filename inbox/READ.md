# Inbox 需求入口

这个文件夹用于临时收集我还没有整理好的想法、需求、选题、问题和灵感。

## 什么时候放到这里？

当我出现以下情况时，先放到 inbox：

1. 有一个短视频选题，但还没判断是否值得做
    
2. 有一个直播主题，但还没确定人群和转化路径
    
3. 有一个 AI 工具想法，但还没形成项目
    
4. 有一段提示词，但还没整理成模板
    
5. 有一个学习问题，但还没归档到 docs / Skill / prompts
    

## 处理规则

inbox 不是长期存放区。

每周至少整理一次，把内容移动到对应位置：

- 选题分析 → Skill 或 docs
    
- 提示词 → prompts
    
- 学习笔记 → docs
    
- 项目想法 → projects
    
- 操作记录 → logs
    

## 我的处理流程

1. 先把想法放进 inbox
    
2. 判断它属于哪一类
    
3. 找已有的 Skill / prompt / docs
    
4. 处理完成后移动到正式目录
    
5. 提交到 GitHub
